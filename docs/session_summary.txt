Current goal
============

The current priority is C1, the professor-requested relation-preserving
baseline translator in `translator_bs.ml`.

C1 must generate:

- `step      : Config -> StepConf`
- `step-pure : InstrTerminals -> StepPureConf`
- `step-read : Config -> StepReadConf`
- `steps     : Config -> StepsConf`

`steps` must be unary:

```maude
steps(C) => C''
  if step(C) => C' /\ steps(C') => C'' .
```

Other requirements:

- use `_ ; _` for config syntax;
- use WT/`WasmTerminal`, not RT/`RecordTerminal`;
- do not add `dstep`, `focused-step`, or `mc` to C1;
- do not hard-code benchmark-specific behavior in `translator_bs.ml`;
- keep C2 optimizations for `translator.ml`, not C1.

Current implementation state
============================

Files:

- `translator_bs.ml`: C1 translator.
- `output_bs.maude`: generated C1 output.
- `wasm-exec-bs.maude`: Fibonacci harness over C1.
- `translator.ml`: future C2 path.

Implemented:

- `Step`, `Step_pure`, `Step_read`, and `Steps` are generated separately.
- `Step/pure` uses `step-pure`.
- `Step/read` uses `step-read`.
- `Steps/trans` uses unary `steps`.
- Configs use `_ ; _`.
- Optional `?` binders generate `eps` variants generally.
- `Step/ctxt-instrs` avoids empty recursive focus.
- `mc` has been removed from the main baseline harness path.
- The hand-written executable `$invoke` footer was removed from
  `translator_bs.ml`. Only the SpecTec-generated `$invoke` remains in
  `output_bs.maude`.
- Generic type-iteration lowering for `rollrt`/`unrollrt`/`rolldt` was fixed.
  The generated output no longer leaves `FREE-UNROLLRT0-I` /
  `FREE-ROLLRT0-I` / `FREE-ROLLDT0-I` for the `(i<n)` iterator in those rules.

Current Fibonacci state
=======================

`WASM-FIB-BS` is a hand-written harness, not generated semantics.

`fib-config(N)` currently reduces to:

```maude
fib-store ; empty-frame ;
  N i32.const 0 i32.const 1 ref.func 0 call_ref fib-type
```

This is closer to the desired call path than the old direct `fib-body` shortcut,
but it still does not execute past the first `call_ref`.

Narrowed blocker
================

The current blocker is before model checking.

The previous failing path was:

```maude
step-read-call-ref-func
  -> $expanddt(value('TYPE, fib-funcinst))
  -> $unrolldt(fib-type)
```

That type-unrolling path has now been fixed for the fib type:

```maude
red in WASM-FIB-BS : $unrollrt(...) .
red in WASM-FIB-BS : $expanddt(fib-type) .
```

The new minimum failing term is:

```maude
red in WASM-FIB-BS : CTORLEA1(CTORSA0) .
```

This stack-overflows. Because `fib-loop-body` contains
`CTORRELOPA2(CTORI32A0, CTORLEA1(CTORSA0))`, `fib-loop-body`,
`value('CODE, fib-funcinst)`, and then `step-read(fib-config(...))` also
overflow. The next fix should target generated numeric operator constructor /
membership executability, generally and not as a Fibonacci-specific patch.
