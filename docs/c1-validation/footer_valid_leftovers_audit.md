# Footer `= valid` Leftovers Audit

Updated: 2026-05-20

## Scope

This audit classifies the footer `eq` / `ceq ... = valid` statements that remained after strict validation lowering reached:

```text
281 strict source-rule targets
-> 281 primary Maude rl/crl rules
```

Strict C1 policy does not allow duplicate source-rule footer equations. If a source relation rule already has a primary generated `rl` / `crl`, a footer equation for the same source fact is non-isomorphic duplicate scaffolding.

## Rationale

`Expand`, `Num-ok`, and singleton `Val-ok` are real SpecTec source relation rules. They are already emitted by generic RelD lowering as primary Maude `crl` rules, so the footer equations were duplicate source-rule scaffolding.

The remaining sequence-shaped `Val-ok` equations are different: SpecTec has singleton `Val_ok`, while some execution premises use pointwise iteration over `val*` / `valtype*`. The footer equations provide an executable list-lifting shortcut for that harness/prelude use. They are not strict C1-final and should eventually be moved out of the strict core or replaced by a principled C2 execution layer.
